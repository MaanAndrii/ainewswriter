# AI Newswriter

Інструмент для підготовки публікацій українського онлайн-медіа на базі AI. Приймає вхідний матеріал будь-якою мовою і повертає готові заголовки, ліди, статтю та Facebook-допис у форматі JSON.

## Можливості

- **Рерайт і підготовка новин** — вхідний матеріал будь-якою мовою перетворюється на готову публікацію українською
- **Кілька форматів виводу** — заголовки, ліди, стаття, Facebook-допис
- **Гнучкі параметри обробки:**
  - Глибина рерайту (0–3): мінімальні правки → повна переробка структури
  - Тональність: нейтральна, інтригуюча, емоційна, SEO
  - Додаткові інструкції до кожного запиту
  - Веб-пошук для актуалізації фактів (Anthropic і Gemini)
  - Органічне вписування джерела в перший абзац
- **7 AI-провайдерів** — Anthropic, xAI, Google Gemini, Mistral, OpenAI, DeepSeek, Groq
- **Асинхронна обробка** — запит відправляється у фоновий воркер, інтерфейс опитує статус без блокування
- **Адмін-панель** — управління моделями, ключами, промтами, статистика, імпорт/експорт
- **Детальні логи** — вартість, токени, кеш-статус, час відповіді по кожному запиту
- **Prompt caching** — кешування системного промту для Anthropic (знижує вартість)

## Швидкий старт

### PHP вбудований сервер (розробка)

```bash
git clone <repo-url> ainewswriter
cd ainewswriter
php -S localhost:8080
```

Відкрийте `http://localhost:8080` у браузері.

Перейдіть до адмін-панелі (`/admin`), увійдіть з паролем за замовчуванням **`change-me-now`**, одразу змініть пароль у вкладці **Безпека** та введіть API-ключі у вкладці **Налаштування AI**.

### Продакшн (nginx + PHP-FPM)

```bash
sudo apt install -y git
git clone <repo-url> ainewswriter
cd ainewswriter
sudo bash install.sh
```

`git` потрібен до клонування — на свіжому Raspberry Pi OS його може не бути. Скрипт автоматично встановлює решту пакетів, налаштовує nginx і PHP-FPM, виставляє права та перезапускає сервіси. Після завершення виводить локальну IP-адресу та нагадування про перший вхід.

Детальний опис кожного кроку — у [RASPBERRY_PI_SETUP.md](RASPBERRY_PI_SETUP.md).

## Підтримувані AI-провайдери

| Провайдер | Формат API |
|-----------|------------|
| Anthropic (Claude) | Messages API |
| xAI (Grok) | OpenAI-compatible |
| Google Gemini | Generative Language API |
| Mistral | OpenAI-compatible |
| OpenAI | Chat Completions API |
| DeepSeek | OpenAI-compatible |
| Groq | OpenAI-compatible |

API-ключі вводяться в адмін-панелі (вкладка **Налаштування AI**) і зберігаються автоматично.

### Додавання моделей

Моделі налаштовуються в адмін-панелі. При додаванні вказуються:

| Поле | Опис |
|------|------|
| ID моделі | Ідентифікатор на боці провайдера (напр. `claude-sonnet-4-6`) |
| Назва | Відображається у випадаючому списку UI |
| Провайдер | Один з 7 підтримуваних |
| Ціна вхід / вихід | Вартість у $ за 1M токенів |
| Веб-пошук | Увімкнути опцію для цієї моделі (підтримується: Anthropic, Gemini) |
| Тайм-аут | Час очікування відповіді в секундах (30–600, за замовч. 120) |

|-----------|-----------|------------|

## Адмін-панель

**URL:** `/admin`  
**Пароль за замовчуванням:** `change-me-now` — змінити одразу після першого входу у вкладці **Безпека**.

### Вкладки

#### Налаштування AI
- Введення та оновлення API-ключів (відображаються замасковано)
- Додавання, редагування, видалення AI-моделей
- Зміна порядку моделей перетягуванням

#### Промти і параметри
- Редагування системного промту
- Ліміти генерації: кількість заголовків, лідів, максимальна довжина статті та Facebook-допису, мін/макс довжина ліду
- Профілі промту: тональності, глибина рерайту, стилі Facebook, правила вписування джерела

#### Статистика
- Загальна кількість запитів, витрати, токени
- Розбивка по моделях і денна динаміка (останні 14 днів)

#### Безпека
- Зміна пароля адмін-панелі
- Зміна пароля log viewer (за замовчуванням збігається з паролем адміна)

#### Логи
- Останні запити з фільтрацією по даті
- Вартість, токени, кеш-статус, час відповіді
- Перегляд сирих відповідей API (останні 5)

#### Імпорт / Експорт
- **Експорт** — поточні налаштування у JSON (без API-ключів)
- **Імпорт** — відновлення з резервної копії (моделі, промти, постобробка, ключі)

## Налаштування промтів

Файл `prompts.json` задає стандартні значення. Усі ліміти та тексти редагуються в адмін-панелі без зміни файлу.

### Плейсхолдери юзер-промту

| Плейсхолдер | Значення |
|-------------|----------|
| `{{headlines_count}}` | Кількість заголовків |
| `{{leads_count}}` | Кількість лідів |
| `{{lead_min_chars}}` / `{{lead_max_chars}}` | Межі довжини ліду |
| `{{article_max_chars}}` | Максимум символів статті |
| `{{facebook_max_chars}}` | Максимум символів Facebook-допису |
| `{{tone_short}}` | Коротка інструкція тональності |
| `{{depth_short}}` | Коротка інструкція глибини рерайту |
| `{{source_ref}}` | Назва джерела для органічного вписування |
| `{{fb_style_rule}}` | Правило стилю Facebook-допису |

## Розгортання на Raspberry Pi

Детальна покрокова інструкція — у файлі [RASPBERRY_PI_SETUP.md](RASPBERRY_PI_SETUP.md).

**Короткий огляд:**

1. Встановіть пакети:
   ```bash
   sudo apt install -y nginx php-fpm php-curl php-sqlite3 git
   ```

2. Клонуйте репозиторій у `/var/www/ainewswriter`

3. Налаштуйте nginx (конфіг з таймаутами для reasoning-моделей):
   ```nginx
   location / {
       try_files $uri $uri/ /index.php?$query_string;
   }
   location ~ \.php$ {
       fastcgi_pass unix:/run/php/php8.4-fpm.sock;
       fastcgi_read_timeout 300;
       fastcgi_send_timeout 300;
       fastcgi_index index.php;
       include fastcgi.conf;
   }
   ```
   > `fastcgi.conf` (не `snippets/fastcgi-php.conf` і не `fastcgi_params`) — він містить коректно налаштований `SCRIPT_FILENAME` без проблемного `try_files $fastcgi_script_name =404`.

4. Збільшіть таймаут PHP-FPM (`/etc/php/8.4/fpm/pool.d/www.conf`):
   ```ini
   request_terminate_timeout = 300
   ```

5. Відкрийте адмін-панель, увійдіть з паролем `change-me-now`, змініть пароль і введіть API-ключі.

> **Чому 300 секунд?** Reasoning-моделі можуть думати 60–180 сек. Без збільшення таймауту nginx поверне `504 Gateway Timeout` до того як модель відповість.

## Безпека

- API-ключі та паролі зберігаються у `.env.local` (gitignored, права `0600`), керуються через адмін-панель
- Файл налаштувань `settings_store.php` недоступний через веб (заблокований роутером)
- Сесійна авторизація для адмін-панелі та log viewer
- CORS-обмеження на рівні API
- Вхідний промт обмежений 30 000 символів
- Система розрахована на роботу у закритій локальній мережі — не публікуйте адмін-панель назовні

## Залежності

**PHP (сервер):**
- PHP 8.1+ (тестувалось на 8.4)
- Розширення: `php-curl`, `php-sqlite3`

**Веб-сервер:**
- nginx + PHP-FPM (рекомендовано)
- або `php -S localhost:PORT` (розробка)

## Діагностика

### Перевірка nginx і PHP-FPM

```bash
sudo nginx -t
sudo tail -n 100 /var/log/nginx/error.log
sudo journalctl -u php8.4-fpm -n 100 --no-pager
```

### Перевірка синтаксису PHP

```bash
cd /var/www/ainewswriter
find . -maxdepth 3 -name "*.php" | xargs -n1 php -l
```

### Типові помилки

| Помилка | Причина | Вирішення |
|---------|---------|-----------|
| `HTTP 504` | Gateway timeout — reasoning-модель думає довше ніж таймаут nginx | Збільшіть `fastcgi_read_timeout` до 300 секунд |
| `Порожня відповідь від API` | Модель повернула порожній або заблокований контент | Спробуйте іншу модель або перефразуйте запит |
| `Failed to fetch` | Тимчасовий збій мережі між браузером і сервером | Інтерфейс автоматично робить 3 повторні спроби |
| `Немає доступних моделей` | `settings_store.php` порожній або пошкоджений | Додайте модель через адмін-панель |
| Адмін-панель не зберігає | `settings_store.php` або `storage/` недоступні для запису | `sudo chown -R www-data:www-data /var/www/ainewswriter` |
| `exec() вимкнено` | Хостинг заблокував `exec()` | Потрібен VPS або self-hosted сервер |
